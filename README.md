* **IDE**: VS Code (Go extension)

* **Build Tool**: Makefile
* **VCS**: Git

---

## 2. Структура проєкту
Проєкт організований відповідно до Standard Go Project Layout:
* `cmd/app/main.go` — точка входу (ініціалізація та виклик функцій).
* `internal/math.go` — основна логіка проєкту (функції Add та Divide).
* `internal/math_test.go` — Unit-тести (Table-driven tests).
* `Makefile` — автоматизація рутинних завдань.
* `.golangci.yml` — конфігурація статичного аналізу коду.

---

## 3. Як працювати з проєктом

### Форматування коду
Для автоматичного форматування коду згідно зі стандартами Go:
```bash
go fmt ./...
Запуск тестів
Використовуються табличні тести з перевіркою на стан гонитви (Race Detector):

Bash
go test -v -race ./internal/...
Статичний аналіз (Лінтер)
Для перевірки якості коду та пошуку помилок (errcheck, gosimple, unused, revive, staticcheck):

Bash
.\golangci-lint.exe run ./...
Збірка бінарного файлу
Компіляція виконуваного файлу Windows у папку /bin:

Bash
go build -o bin/app.exe ./cmd/app/main.go
